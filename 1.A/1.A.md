# Exercise 1.A: Moving Averages Tool

## Purpose

This exercise is designed to strengthen your **programming fundamentals** by building a practical financial analysis tool. You will learn to:
- Load and manipulate real-world stock market data using Pandas
- Perform time-series calculations and data transformations
- Create professional visualizations using Matplotlib
- Handle edge cases and different data scenarios
- Write clean, reusable code for financial analysis

This exercise bridges computer science fundamentals with practical AI applications, demonstrating how technical indicators drive trading strategies and algorithmic decision-making.

---

## Objective

Build a Python tool that accepts any stock ticker from the S&P 500 and generates a visualization comparing its **50-day moving average (MA50)** and **200-day moving average (MA200)**.

---

## Requirements

### Input
- Accept a stock ticker symbol (e.g., "AAPL", "MSFT", "TSLA") as user input
- Load historical price data from the [S&P 500 Dataset](https://www.kaggle.com/datasets/camnugent/sandp500)

### Processing
1. **Data Loading**: Read the CSV file containing S&P 500 historical data
2. **Data Filtering**: Filter data for the selected stock ticker
3. **Moving Average Calculation**:
   - Calculate the 50-day moving average (MA50)
   - Calculate the 200-day moving average (MA200)
   - Handle edge cases (e.g., insufficient data for initial periods)
4. **Data Validation**: Ensure data quality and handle missing values appropriately

### Output
Create a visualization that displays:
1. **Stock Price**: Line plot of closing prices (in a distinct color)
2. **MA50**: Line plot of 50-day moving average (e.g., blue)
3. **MA200**: Line plot of 200-day moving average (e.g., red)
4. **Legend**: Clearly labeled for each line
5. **Axes Labels**: Date on x-axis, price on y-axis
6. **Title**: Display the stock ticker and date range (e.g., "AAPL Moving Averages (2019-2024)")
7. **Grid**: Optional but recommended for readability

## Hints & Tips

1. **Moving Average Calculation**: Use `pd.DataFrame.rolling(window=50).mean()`
2. **Data Structure**: Keep your data organized with columns: `Date`, `Close`, `MA50`, `MA200`
3. **Edge Cases**: The first 199 days won't have a valid MA200—decide how to handle this (drop, keep as NaN, or plot only valid data)
4. **Financial Logic**: In trading, when MA50 crosses above MA200, it signals a bullish trend (Golden Cross). When it crosses below, it's bearish (Death Cross). Consider adding annotations for these events as an **bonus feature**.
