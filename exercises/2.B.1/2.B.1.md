# Exercise 2.B.1: Decision Trees vs. SVM for Customer Churn Prediction

## Purpose

This exercise applies supervised learning to **Subscriber Retention**, a critical business challenge. You will compare **Decision Trees** and **Support Vector Machines (SVM)** using a rich, multi-dimensional Kaggle dataset. You will learn to:
- Process high-cardinality categorical data (One-Hot Encoding)
- Handle class imbalance (more customers stay than leave)
- Compare "rule-based" business logic (Trees) with "feature-space" isolation (SVM)
- Visualize the trade-offs between interpretability and high-dimensional accuracy

---

## Objective

Build a Python script to predict which customers are likely to churn (cancel their subscription) based on the **Telco Customer Churn** dataset. You will implement both a **Decision Tree** and an **SVM** and compare their ability to identify "at-risk" customers.

---

## Requirements

### Input
- The [Kaggle Telco Customer Churn Dataset](https://www.kaggle.com/datasets/blastchar/telco-customer-churn) (standard `WA_Fn-UseC_-Telco-Customer-Churn.csv`)
- Key Features: MonthlyCharges, TotalCharges, Tenure, Contract Type, Payment Method, Technical Support, etc.

### Processing
1. **Data Preprocessing**:
   - Convert `TotalCharges` to numeric (handle empty strings).
   - Encode categorical features using `get_dummies` or `OneHotEncoder`.
   - Map the target `Churn` (Yes/No) to binary ($1/0$).
   - Perform a stratified 80/20 train/test split.
2. **Feature Scaling**:
   - Apply `StandardScaler` to all features, which is mandatory for the **SVM**.
   - Train the **Decision Tree** on the same scaled data for a fair comparison.
3. **Model Implementation**:
   - **Decision Tree**: Use `DecisionTreeClassifier` with `class_weight='balanced'` and `max_depth=6`.
   - **SVM**: Use `SVC` with an **RBF kernel** and `probability=True` to enable ROC analysis.

### Output
1. **Metrics Report**: Compare Accuracy, F1-Score, and the **Area Under the ROC Curve (AUC)**.
2. **Visualizations**:
   - Plot the **ROC Curve** for both models on a single chart.
   - Plot a **Confusion Matrix** to see if the models are biased toward the majority class.
3. **Business Insight**: List the top 3 features the Decision Tree identified as precursors to churn (e.g., Month-to-month contracts).

## Hints & Tips

1. **Class Imbalance**: Churn datasets are usually imbalanced. Using `class_weight='balanced'` helps the models not just guess "No Churn" for everyone.
2. **TotalCharges Error**: Watch out for empty strings in `TotalCharges` when converting from Object to Float.
3. **Tree Depth**: A deep tree will overfit; keep `max_depth` under 7 for this dataset to maintain generalization.
