# Deep Multi-Layer Perceptron Architectures
**Difficulty Level:** Phase 3 (Intermediate) | **Time Limit:** 3 Hours

## 1. Background / Scenario
You are a deep learning architectural researcher testing the validity of synthetic manifolds against non-linear Multi-Layer Perceptron frameworks. Manual numerical derivatives generated using `numpy` loops are far too inefficient and prone to implementation bugs for expansive datasets. You must utilize an industry-standard computational framework to map deep learning metrics natively. 

## 2. Problem Statement
Operating over the Synthetic Moons dataset, design a 3-layer Multi-Layer Perceptron deploying PyTorch's high-level `nn.Module` arrays. Optimize non-linear decision boundaries through sequential `nn.ReLU` and `nn.CrossEntropyLoss` metrics. Execute the mathematical backpropagation steps using `autograd` natively, bypassing any from-scratch chain-rule gradient formulations.

## 3. Dataset Description
**Reference:** [Synthetic Moons Dataset](https://scikit-learn.org/stable/modules/generated/sklearn.datasets.make_moons.html)

The data comprises synthetically intertwined topological half-circles, mapping nonlinear spatial points with explicit Gaussian noise perturbations generated by `sklearn.datasets.make_moons()`. Features are inherently low-dimensional but notoriously difficult for linear discriminators to separate effectively.

## 4. Subtasks & Point Distribution
* **Task 4.1: Dataset Tensors & Iterators (30 pts):** Instantiate explicit PyTorch `TensorDataset` and `DataLoader` sequences mapping the Numpy generation outputs explicitly onto standard FloatTensors. 
* **Task 4.2: High-Level `nn.Module` Logic (40 pts):** Construct precise neural layers using `nn.Sequential` or custom class inheritance wrapping three standard `nn.Linear` fully-connected transformations interleaved with spatial `nn.ReLU` non-linearities. 
* **Task 4.3: Optimal Forward & Backpropagation (30 pts):** Detail an exact training loop enforcing dynamic metric mapping over PyTorch gradients iteratively bounding `loss.backward()` correctly integrating step updates natively parameterized using `torch.optim.SGD` or `torch.optim.Adam`.

## 5. Constraints & Technical Rules
* **Libraries:** You are strictly permitted to deploy PyTorch, Scikit-Learn, and NumPy arrays for this constraint, isolating manual math matrices backwards completely.
* **Execution Constraint:** Writing explicit backward passes using iterative `for` loops tracking mathematical partial derivatives manually is forbidden. Use `loss.backward()`.
* **Modeling Logic:** Your structural model must systematically map multi-layer configurations tracking epochs and partial fitting parametrically via PyTorch `autograd` and built-in loss calculations (`nn.CrossEntropyLoss` or `nn.BCELoss`).

## 6. Evaluation Criteria
Structural validation constraints enforce correct separation of the synthetic moon geometries mapping a strict non-linear hyperplane explicitly visualized across hold-out matrices. Weight updates mapping dynamically on `autograd` must stabilize test array probabilities explicitly within standard topological time limits.

## 7. Deliverables
* `pipeline.py`: A cohesive python script implementing the `nn.Module` mapping and dataset prep functionally.
