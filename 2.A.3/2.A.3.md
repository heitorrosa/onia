# Exercise 2.A.3: Regularization and Feature Sparsity with Elastic Net

## Purpose

This exercise explores the **Regularization** aspect of linear models to solve the problem of **Overfitting** and **Multicollinearity**. You will learn to:
- Combine L1 (Lasso) and L2 (Ridge) penalties using **Elastic Net**
- Use regularization as a tool for automatic feature selection (Sparsity)
- Tune multiple hyperparameters ($C$ and $l1\_ratio$) simultaneously
- Understand the Bias-Variance tradeoff in the context of high-dimensional datasets

This exercise demonstrates how regularization transforms a standard linear model into a robust tool capable of handling hundreds of features while identifying the most important ones.

---

## Objective

Build a binary classifier using **Regularized Logistic Regression** (Elastic Net) to classify breast cancer tumors as *Malignant* or *Benign*. You will analyze how the ratio of L1/L2 penalties affects the number of "selected" features (non-zero weights).

---

## Requirements

### Input
- The [Breast Cancer Wisconsin (Diagnostic) Dataset](https://archive.ics.uci.edu/ml/datasets/Breast+Cancer+Wisconsin+(Diagnostic)) (loadable via `sklearn.datasets.load_breast_cancer`)
- 30 high-dimensional features (Mean radius, mean texture, etc.)

### Processing
1. **Stratified Data Splitting**: Ensure balanced representation of classes in training and testing sets.
2. **Feature Standardization**: Scaling is mandatory here, as different feature magnitudes would unfairly bias the regularization penalties.
3. **Model Implementation**:
   - **Logistic Regression with Elastic Net**: Use `penalty='elasticnet'`, `solver='saga'`, and vary the `l1_ratio` (0 = Ridge, 1 = Lasso, 0.5 = Equal mix).
4. **Hyperparameter Tuning**:
   - Use `GridSearchCV` or `RandomizedSearchCV` to find the optimal combination of `C` (inverse regularization strength) and `l1_ratio`.

### Output
1. **Regularization Path Plot**: A chart showing how the weights of individual features shrink to zero as regularization strength ($1/C$) increases.
2. **Sparsity Report**: Calculate and print the number of features that were effectively "zeroed out" by different L1-ratios.
3. **Confusion Matrix**: Visualize the classification results for the final optimized model.

## Hints & Tips

1. **SAGA Solver**: Remember that for `penalty='elasticnet'`, you must use `solver='saga'` in Scikit-learn's `LogisticRegression`.
2. **L1 vs L2**: L1 (Lasso) is "aggressive" and forces weights to 0 (sparsity), while L2 (Ridge) merely shrinks them. Elastic Net allows you to find a middle ground.
3. **C parameter**: In Scikit-learn, `C` is the *inverse* of regularization strength. Small `C` = strong regularization; large `C` = weak regularization (closer to OLS).
