# Exercise 2.A.2: Optimizing Gradient Descent for Binary Classification

## Purpose

This exercise focuses on the **Optimization** phase of supervised learning. You will move beyond simple model fitting to understand the underlying engines of machine learning: **Gradient Descent (GD)** and **Learning Rate (LR)** tuning. You will learn to:
- Understand the trade-offs between Batch, Stochastic (SGD), and Mini-batch Gradient Descent
- Experiment with different Learning Rate strategies (Constant vs. Adaptive Schedulers)
- Visualize convergence behavior and identify signs of divergence or slow learning
- Implement early stopping based on validation loss curves

This exercise demonstrates how the choice of optimizer and learning rate can drastically affect the stability and performance of linear classification models.

---

## Objective

Build a binary classifier using **Logistic Regression** (optimized via SGD) to predict the likelihood of diabetes onset. You will specifically analyze how the learning rate and optimization strategy impact the model's accuracy and convergence speed.

---

## Requirements

### Input
- The [Pima Indians Diabetes Dataset](https://www.kaggle.com/datasets/uciml/pima-indians-diabetes-database) (e.g., `diabetes.csv`)
- Features: Pregnancies, Glucose, Blood Pressure, Skin Thickness, Insulin, BMI, Diabetes Pedigree Function, Age

### Processing
1. **Data Cleaning**: Handle missing values (encoded as 0 in columns like BMI/Glucose) using median imputation or deletion.
2. **Standardization**: Use `StandardScaler` to ensure all features are on the same scale, which is critical for Gradient Descent convergence.
3. **Model Implementation**:
   - **SGDClassifier**: Use scikit-learn's `SGDClassifier` with `loss='log_loss'` (formerly `log`) for logistic regression.
   - **Optimizer Comparison**: Compare **Batch GD** (using all training data per step) vs. **Stochastic GD** (one sample per step).
4. **Learning Rate Optimization**:
   - Test at least three LR strategies: `constant`, `invscaling`, and `adaptive`.
   - Experiment with different initial learning rates (e.g., $10^0$, $10^{-2}$, $10^{-4}$).

### Output
1. **Convergence Plots**: Generate a "Loss vs. Epochs" or "Loss vs. Iterations" line chart showing how different learning rates converge.
2. **Metrics Report**: A classification report (Precision, Recall, F1) for the best-performing configuration.
3. **Execution Summary**: A brief comparison of execution time vs. accuracy for Batch vs. SGD.

## Hints & Tips

1. **SGD vs. GradientDescent**: Note that `LogisticRegression(solver='lbfgs')` is a second-order optimizer, while `SGDClassifier` uses first-order Gradient Descent. Use `SGDClassifier` for this exercise to see the GD behavior.
2. **Warm Start**: Use `warm_start=True` in `SGDClassifier` inside a loop to manually track loss after each epoch.
3. **Divergence**: If the loss becomes `NaN` or increases rapidly, your learning rate is likely too high.
