# Exercise 2.A.1: Multiclass Classification with Softmax Regression

## Purpose

This exercise focuses on **Supervised Learning** for classification tasks. You will transition from predicting continuous values (regression) to categorizing data into discrete classes. You will learn to:
- Handle multiclass tabular datasets (Iris dataset)
- Understand the **Softmax function** and its role in Multinomial Logistic Regression
- Implement feature scaling for gradient-based convergence
- Evaluate classification performance using Confusion Matrices and F1-scores

This exercise demonstrates how machine learning models can be used to identify biological patterns and automate species classification.

---

## Objective

Build a Python script that uses **Softmax Regression** (Multinomial Logistic Regression) to classify Iris flowers into one of three species (*Setosa*, *Versicolor*, or *Virginica*) based on sepal and petal measurements.

---

## Requirements

### Input
- The [Iris Dataset](https://www.kaggle.com/datasets/uciml/iris) (available directly via `sklearn.datasets.load_iris`)
- Features: Sepal length, Sepal width, Petal length, Petal width

### Processing
1. **Exploratory Data Analysis (EDA)**: Create a pairplot or scatter plot to visualize feature relationships and class separability.
2. **Data Splitting**: Implement a stratified train/test split (e.g., 80% training, 20% testing) to ensure balanced class representation.
3. **Feature Scaling**: Apply `StandardScaler` to normalize the measurements.
4. **Model Implementation**:
   - **Logistic Regression**: Configure for multinomial classification (`multi_class='multinomial'`) using the `lbfgs` solver.
   - **Regularization**: Use L2 regularization (penalty) to prevent overfitting.

### Output
1. **Metrics Report**: Print Accuracy, Precision, Recall, and F1-score (weighted and per-class).
2. **Confusion Matrix**: A heatmap visualization showing correct vs. incorrect classifications across all species.
3. **Decision Boundaries**: (Optional/Bonus) A 2D plot showing the model's decision regions using the two most important features.

## Hints & Tips

1. **Softmax vs. Sigmoid**: Remember that Softmax is the generalization of the Sigmoid function for $K > 2$ classes, outputting a probability distribution that sums to 1.
2. **Stratification**: Use `stratify=y` in `train_test_split` to avoid getting a test set that lacks one of the species.
3. **One-vs-Rest (OvR)**: Understand the difference between the 'multinomial' (True Softmax) approach and the 'ovr' (One-vs-Rest) approach often used in binary classifiers.
