# Exercise 2.B: The "Bull or Bear" Classifier (Decision Trees vs. SVM)

## Purpose

This exercise explores **Supervised Learning** for binary classification in volatile environments. You will compare two fundamentally different algorithms: **Decision Trees** (rule-based) and **Support Vector Machines** (geometric margin-based). You will learn to:
- Transform continuous financial data into binary signals (Bull vs. Bear)
- Understand the impact of kernel selection in SVMs
- Visualize decision boundaries and tree structures
- Evaluate model robustness using ROC Curves and AUC

---

## Objective

Build a Python script that classifies whether a stock's price will rise by more than 2% the following day ("Bull") or not ("Bear"). You will implement both a **Decision Tree** and an **SVM** and compare their predictive power.

---

## Requirements

### Input
- [Stock Market Dataset for Predictive Analysis](https://www.kaggle.com/datasets/s3programmer/stock-market-dataset-for-predictive-analysis)
- Features: Open, High, Low, Close, Volume

### Processing
1. **Label Generation**: Create a target variable `y` where `1` (Bull) if `(Close_tomorrow - Close_today) / Close_today > 0.02`, else `0` (Bear).
2. **Feature Engineering**: Include technical indicators like RSI or Moving Averages to provide more context than raw prices.
3. **Data Splitting**: Use a chronological split (80/20) to respect the time-series nature of the data.
4. **Model Implementation**:
   - **Decision Tree**: Use `DecisionTreeClassifier` with pruning (`max_depth`) to prevent overfitting.
   - **SVM**: Use `SVC` with an RBF kernel; apply `StandardScaler` as SVMs are sensitive to feature scales.
5. **Hyperparameter Comparison**: Observe how changing the tree depth or SVM `C` parameter affects the results.

### Output
1. **Classification Report**: Precision, Recall, and F1-score for both models.
2. **ROC Curve**: A single plot comparing the ROC curves of the Tree vs. SVM.
3. **Tree Visualization**: Export or plot the top levels of the Decision Tree to see which features are the primary "splitters."

## Hints & Tips

1. **Imbalanced Classes**: Market jumps of >2% might be rare. Check your class distribution; if it's highly imbalanced, consider using `class_weight='balanced'`.
2. **SVM Complexity**: SVMs can be slow on very large datasets. If the script hangs, use a smaller subset of the data or a linear kernel first.
3. **Overfitting**: Decision Trees are prone to memorizing the training set. Always limit `max_depth` or `min_samples_leaf`.
