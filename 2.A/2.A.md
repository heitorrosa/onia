# Exercise 2.A: Price Prediction with Regularized Linear Regression

## Purpose

This exercise is designed to introduce **Supervised Learning** principles, focusing on regression and the importance of **Regularization**. You will learn to:
- Prepare time-series data using "sliding window" feature engineering
- Understand the mathematical differences between **L1 (Lasso)** and **L2 (Ridge)** regularization
- Implement predictive models to forecast numerical values (Closing Price)
- Use Scikit-learn's pipeline for data scaling and model training
- Evaluate regression performance using metrics like MAE, RMSE, and $R^2$

This exercise demonstrates how foundational machine learning models are used to identify trends and prevent overfitting in noisy financial data.

---

## Objective

Build a Python script that uses **Linear Regression** with regularization to predict the next day's closing price based on a window of the previous **five trading days**.

---

## Requirements

### Input
- Historical price data from the [Financial Market Forecasting Dataset](https://www.kaggle.com/datasets/ziya07/financial-market-forecasting-dataset)
- A selected stock ticker or a specific market index file

### Processing
1. **Feature Engineering**: Create a lagged dataset where each row contains the closing prices of the past 5 days ($t-1$ to $t-5$) to predict the price at $t$.
2. **Data Splitting**: Implement a chronological train/test split (e.g., first 80% for training, last 20% for testing) to avoid data leakage.
3. **Feature Scaling**: Apply `StandardScaler` to ensure the model converges correctly.
4. **Model Implementation**:
   - **Ridge Regression (L2)**: To handle multicollinearity between lagged prices.
   - **Lasso Regression (L1)**: To perform automatic feature selection.
5. **Model Comparison**: Compare the coefficients and performance of both models.

### Output
1. **Prediction Plot**: A line chart comparing the "Actual Prices" vs. "Predicted Prices" on the test set.
2. **Metrics Report**: Print the Mean Absolute Error (MAE) and Root Mean Squared Error (RMSE) for both Ridge and Lasso.
3. **Weight Analysis**: A bar chart visualizing the coefficients (weights) assigned to each of the 5 lagged days.

## Hints & Tips

1. **Lagging Data**: Use `df['Close'].shift(i)` to create the 5 input features.
2. **Data Leakage**: Never use `train_test_split(shuffle=True)` for time-series, as the model would "see" the future during training. Use `TemporalSplit` or manual slicing.
3. **Regularization Strength**: Experiment with the `alpha` parameter in Ridge and Lasso. Small values approach OLS, while large values simplify the model.
