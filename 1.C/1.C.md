# Exercise 1.C: Real-time Market Dashboard

## Purpose

This exercise focuses on **interactive data visualization and event detection**. You will learn to:
- Visualize complex financial patterns (Candlesticks) using Python
- Synchronize multiple plots (Price vs. Volume) in a single dashboard
- Implement statistical logic to identify "volatility spikes"
- Use `Seaborn` and `Matplotlib` to create professional-grade financial reports
- Handle time-series data alignment for multi-axis visualizations

This exercise bridges technical analysis with data science, showing how visual patterns correlate with market behavior and automated trading signals.

---

## Objective

Build a Python tool that generates a comprehensive market dashboard for any stock. The dashboard must display a **Candlestick Chart** for price action and a **Volume Bar Chart** that highlights unusual activity (spikes) to identify high-volatility events.

---

## Requirements

### Input
- Accept a stock ticker symbol (e.g., "AAPL", "TSLA", "BTC-USD")
- Load historical data from the [Stock Market Dataset for Financial Analysis](https://www.kaggle.com/datasets/s3programmer/stock-market-dataset-for-financial-analysis)

### Processing
1. **Data Pre-processing**: Ensure columns `Open`, `High`, `Low`, `Close`, and `Volume` are correctly formatted
2. **Candlestick Logic**: Prepare the data for candlestick plotting (using `mplfinance` or manual Matplotlib patches)
3. **Volatility Detection**:
   - Calculate a moving average of trading volume (e.g., 20-day average)
   - Identify "Spikes": Days where volume is >2x the recent average
4. **Dashboard Layout**: Create a vertically stacked layout where the top plot is price and the bottom is volume

### Output
Create a dashboard that displays:
1. **Candlestick Plot**: Showing daily Open-High-Low-Close (OHLC) values
2. **Volume Bars**: A bar chart below the price chart, sharing the same X-axis (Date)
3. **Spike Highlights**: Color-code or annotate volume bars that represent "High Volatility" events
4. **Moving Averages**: Overlay at least one price moving average (e.g., 20-day EMA) on the candlestick chart
5. **Aesthetics**: Proper titles, gridlines, and a dark/light theme for professional appearance

---

## Hints & Tips

1. **Candlesticks in Python**: While you can build them with `matplotlib.lines`, the `mplfinance` library is the industry standard: `mpf.plot(df, type='candle', volume=True)`
2. **Shared Axis**: Use `plt.subplots(2, 1, sharex=True)` to ensure that zooming or scrolling through dates keeps Price and Volume aligned
3. **Volatility Stats**: A simple "Z-score" or "Ratio to Moving Average" is the best way to mathematically define a "spike" in trading volume
4. **Coloring**: Green for "Close > Open" and Red for "Close < Open" is the standard convention for financial markets
